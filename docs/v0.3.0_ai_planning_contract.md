# v0.3.0 AI 规划契约

## AI 职责边界

AI 可以辅助：

- 解析自然语言学习目标。
- 推荐词书组合和阶段路径。
- 解释建议目标词量。
- 识别词库覆盖缺口。
- 诊断周度和月度执行偏差。
- 提出计划参数调整建议。
- 生成用户可理解的调整说明。

AI 不得：

- 直接写入正式目标、任务、历史或词书启用范围。
- 删除学习欠缺或逾期复习。
- 生成旧版本不存在的具体学习历史。
- 启用未导入具体词条的词书参与排期。
- 无视每日容量强制安排任务。
- 绕过用户确认修改执行计划。

## 输入数据

AI 或本地规则建议服务只接收脱敏结构化输入：

- 当前目标摘要和目标历史摘要。
- 当前词库数量、已启用词书、推荐词书元数据。
- 今日、周度、月度完成摘要。
- 缺口、欠缺、逾期和可行性数量。
- 用户自然语言目标或诊断请求。

不得输入 API Key、完整备份 JSON 或不必要的用户隐私文本。

## 输出类型

结构化输出包括：

- `goal_parse`：自然语言目标解析。
- `book_recommendation`：词书组合推荐。
- `plan_adjustment`：容量、期限、词量或词书范围调整建议。
- `weekly_diagnosis`：周度偏差诊断。
- `monthly_diagnosis`：月度风险诊断。

## 建议结构

建议必须包含：

- 建议 ID。
- 服务模式：`local_rule` 或 `ai_assisted`。
- 建议类型。
- 原始输入摘要。
- 结构化建议参数。
- 理由。
- 预计影响。
- 本地校验状态。
- 不可应用原因。

## 降级逻辑

- 未配置 AI：使用本地规则模式，核心流程完整可用。
- AI 请求失败：返回本地规则建议，并记录失败原因。
- AI 返回非法结构：不应用，展示错误并保留原计划。
- AI 建议超出容量或缺少真实词库支持：标记为不可直接应用。

## 用户确认流程

1. 用户输入目标或请求诊断。
2. AI 或本地规则生成建议。
3. 页面展示建议。
4. 本地系统计算应用后的影响。
5. 页面显示将改变的参数、词量、阶段和词书范围。
6. 用户明确确认。
7. 创建目标版本或计划调整记录。
8. 本地调度器重新生成未来计划。

用户未确认前，不修改正式执行计划。

## API 配置与敏感信息

v0.3.0 默认不接真实 AI API。若实现配置占位：

- API Key 不得硬编码。
- API Key 不得提交 Git。
- API Key 不得写入日志、备份或测试输出。
- 页面必须标明本地保存风险。
- 若无法实现安全保存，只提供接口和本地规则模式，不宣称真实 AI 可用。
