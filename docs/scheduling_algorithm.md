# 调度算法：v0.2.1

> v0.3.0 调度契约见 `docs/v0.3.0_scheduling_contract.md`。当前调度必须继续保持逐词操作与当日结算分离，并新增阶段、周度、月度和可行性复盘输出。

核心实现位于 `src/domain/scheduler.ts`，界面层只调用服务函数，不直接写调度逻辑。

## 新词绑定

1. 根据目标词书范围筛选 `WordItem`。
2. 按 `normalizedWord` 排序后取不超过 `targetRequiredCount` 的真实词条。
3. 为目标词条建立或更新 `WordProgress`。
4. 未完成目标词按优先级排期：
   - `learning_backlog` 先排。
   - `not_started` 后排。
5. 每个每日新词任务生成 `DailyNewWordAssignment`，并绑定具体 `wordId`。

## 词库不足

当 `targetRequiredCount > availableWordCount`：

- 只为真实存在的词条生成任务。
- `inventoryGapCount = targetRequiredCount - availableWordCount`。
- 不伪造缺失的具体单词。
- 不把库存缺口计入用户学习欠缺。
- 调整解释中提示需要补充词书。

## 用户未完成

当用户把具体新词标记为 `skipped` 或 `missed`：

- 对应 `WordProgress.state` 变为 `learning_backlog`。
- 历史任务保留，不删除。
- 重排时优先使用缓冲日，再使用后续有效学习日。
- 如果容量不足，计划状态为 `infeasible`，但不删除待补学词。

## 复习排期

基础节点：

- 新学后第 1 天
- 第 3 天
- 第 7 天
- 第 14 天
- 第 30 天

反馈规则：

- `forgot`：次日复习，`lapseCount + 1`。
- `vague`：缩短下一阶段间隔。
- `known`：进入正常下一阶段。
- `easy`：跳到更远阶段，达到末段后标记掌握。
- `not_completed`：当前任务变为逾期，不删除，后续重新安排。

每次复习写入 `reviewHistory`。

## 联合容量判断

重排时同时计算：

- 每日新词上限。
- 每日复习上限。
- 固定休息日。
- 缓冲日。
- 具体待补学词。
- 逾期复习任务。
- 库存缺口。
- 剩余有效学习日。

不可行状态必须显示最低每日新学量、当前上限和差距。

## 新词书补足缺口

导入词表后：

1. 先进行标准化去重。
2. 重新计算 `availableWordCount` 和 `inventoryGapCount`。
3. 新增且未重复的词进入目标候选词集合。
4. 为新增可用词生成后续 `DailyNewWordAssignment`。
5. 创建 `wordbook_import` 调整日志，记录新增、重复、补足和剩余缺口。

## 历史补录与重复修改

v0.2.1 的事实来源是具体任务状态和 `WordProgress`。同一任务再次修改时以 assignment id 覆盖，不通过数量累加，避免重复累计。v0.1.0 的数量记录只作为 legacy 展示，不参与具体单词完成统计。

## v0.2.1 逐词操作与结算边界

单个词条结果保存只允许影响当前词条及其直接后续任务：

- 新词“已学习”只更新当前 `DailyNewWordAssignment`、`WordProgress`，并为该词建立下一次复习。
- 新词“已掌握”只更新当前词条状态，不影响同日其他开放任务。
- 新词“暂时跳过 / 今日未完成”只将当前词条转为待补学并重排该词，不把同日其他 `planned` 新词计入欠缺。
- 复习“不认识 / 模糊 / 认识 / 很熟悉 / 未完成”只处理当前 `DailyReviewAssignment`。

调度器不得在每次逐词点击后直接把当天未操作任务计入欠缺。原因是逐词点击发生在当天学习过程中，此时其他 `planned` 任务仍是正常开放任务，不代表用户已经放弃或结束当天学习。

## 当日主动结算

用户点击“结束今日学习并安排未完成任务”并确认后，系统才会统一处理当天仍开放的任务：

- 当天仍为 `planned` / `rescheduled` 的新词转为 `missed`，进入待补学。
- 当天仍为 `planned` / `rescheduled` 的复习转为 `overdue`，进入逾期复习。
- 触发一次完整重排。
- 写入 `daily_settlement` 调整日志。

重复结算同一天时，如果没有开放任务，服务直接返回空结算结果，不重复生成任务、不重复增加统计、不重复制造业务调整。

## 跨日自动结算

用户进入后续日期时，系统可以自动结算早于当前目标时区今日的开放任务：

- 只处理过去日期。
- 不处理当前日期仍在进行中的任务。
- 历史开放新词转为待补学。
- 历史开放复习转为逾期复习。
- 写入 `past_due_auto_settlement` 调整日志。
- 多次刷新具备幂等性。

跨日自动结算会保护当前日期已有开放任务的容量，避免历史欠缺任务挤掉今天仍未操作的正常任务。

## 时区

默认使用浏览器本地时区，目标保存 `timezone`。今日任务、复习到期和重排日期统一使用该时区的自然日。
