# v0.3.0 - 智能背词规划与数据一致性正式版

## 版本定位

v0.3.0 是继 v0.1.0 之后的下一次正式发布版本。v0.2.0 和 v0.2.1 均作为未单独发布的开发/技术基线，其有效能力已经承接进入 v0.3.0。

## 关键承接

- v0.1.0 保留为离线数量计划 MVP 基线版本。
- v0.2.0 初步建立单词级学习闭环，但存在逐词操作导致同日任务提前重排的问题，不单独发布。
- v0.2.1 修复逐词操作与当日结算一致性，成为 v0.3.0 技术基线，不再单独发布 Release。
- v0.3.0 完整承接 v0.2.1 的逐词操作、主动结算、跨日结算、旧版备份兼容和词库缺口区分能力。

## 主要新增

- 目标版本历史、阶段计划、每日结算记录、周度复盘、月度复盘、AI 建议记录和 AI 应用记录。
- Dexie v3 数据结构和 v0.3.0 完整备份格式。
- 自然语言目标本地规则建议，用户确认前不会改变正式计划。
- 可解释词书推荐和词书状态区分。
- 基于词书启用顺序、阶段角色、基础补齐、核心标签和学习状态的具体单词选择优先级。
- 今日任务、目标与阶段、长期计划、词书与词库、学习历史和数据设置页面整理。
- 词条学习历史、复习历史和重点遗忘词标识。
- 周度和月度复盘数据，以及不可行和风险数量解释。

## 正式发布前关键修复

- `availableWordCount` 与 `enabledWordCount` 语义和逻辑分离：前者统计当前全部可用去重词，后者只统计当前目标已启用并纳入计划计算的词；导入未启用词书只增加可用词量，不会改写用户已启用范围。
- AI 建议应用记录与普通手动修改行为分离：只有显式应用某条 AI/本地规划建议并保存时才写入 `AIAdviceApplicationRecord`；后续普通手动修改不会重复累积应用记录。
- 新增 tag 自动发布工作流 `.github/workflows/release-on-tag.yml`：推送 `v*` tag 后在 GitHub Actions 中重新安装依赖、运行测试、构建，并使用 `github.token` 创建正式 GitHub Release。

## AI 能力边界

当前版本默认使用本地规则模式，不默认调用真实 AI API。AI 服务结构已预留，建议结果会被本地校验，并且只有用户确认后才会写入目标版本和 AI 应用记录。

## 暂未包含

- 真实 AI API 默认接入。
- 发音、例句、拼写、听力、登录、云同步。
- 自动抓取商业完整词书。
