# v0.4.0 动态进度恢复算法

## 目标

v0.4.0 的动态恢复不是简单把今天少学的任务堆到明天，而是在总目标词数和截止日期不变的前提下，把未完成的新词与逾期复习平滑安排到后续有效学习日。

算法必须继续满足 v0.3.0 约束：

- 新词任务和复习任务都绑定具体 `wordId`。
- 未完成的新词任务不得删除或伪造成已完成。
- 复习历史不得因重排而丢失。
- 词表缺口不计为用户学习失败。
- 系统不得偷偷延长截止日期或减少总目标。

## 核心概念

- 本期目标词数：当前目标版本确认的 `targetRequiredCount`。
- 截止日期：当前目标版本确认的 `deadline`。
- 有效学习日：从重排日期到截止日期之间，排除固定休息日后的日期。
- 原计划每日任务：首次计划或上次正式重排前，每日安排的新词与复习。
- 今日实际完成：用户逐词完成的新词数与复习数。
- 今日未完成任务：当日结算时仍开放的新词和复习。
- 历史待补学任务：已分配具体单词但被跳过、未完成或结算为未完成的新词。
- 历史逾期复习任务：被用户标记未完成或结算为逾期的复习。
- 未来剩余任务：目标中尚未完成的新词、待补学新词和未来复习。
- 未来每日可承受范围：每日新词硬上限、每日复习硬上限和总负荷阈值。
- 当前计划状态：正常、偏高、超限、无法按期完成或已完成。

## 欠缺任务计算

今日少背或缺课时，事实来源必须来自任务状态：

1. 用户点击“结束今日学习并安排未完成任务”。
2. 服务层把当日仍为 `planned` / `rescheduled` 的新词转为 `missed`。
3. 对应 `WordProgress.state` 转为 `learning_backlog`。
4. 当日仍开放的复习转为 `overdue`，并写入复习历史。
5. 调度器只统计这些明确状态，不因为日期早于 `asOfDate` 就自动制造欠缺。

少完成一部分时，只把未完成的具体 `wordId` 纳入待补学。已经完成的新词继续保留学习事实，并照常生成复习。

## 负荷模型

v0.4.0 引入统一负荷模型：

```text
总负荷 = 新词任务数 + 待补学新词数 + (复习任务数 + 逾期复习数) * 复习权重
```

默认复习权重：

```text
reviewWeight = 0.6
```

该权重表示一次复习通常比一次新学更轻，但仍会占用注意力。它必须集中定义，后续可以调整；界面不得把复习和新词完全混为同一种任务，但压力判断必须同时参考两者。

压力状态：

- 休息：固定休息日。
- 轻松：总负荷低于舒适负荷的 70%。
- 正常：总负荷不超过舒适负荷。
- 偏高：总负荷超过舒适负荷，但仍不超过硬上限。
- 超限：新词数、复习数或总负荷超过硬上限。

默认阈值：

```text
舒适负荷 = 每日新词上限 + 每日复习上限 * reviewWeight * 0.75
硬上限负荷 = 每日新词上限 + 每日复习上限 * reviewWeight
```

## 平滑分摊规则

重排未来新词时，算法按以下顺序处理：

1. 保留所有历史已完成任务、学习历史和复习历史。
2. 收集待补学新词，并保持原 `wordId`。
3. 找出从 `asOfDate` 到 `deadline` 的未来有效学习日。
4. 计算每个未来日期的已有负荷，包括原计划新词、已有复习和逾期复习。
5. 按日期负荷从低到高寻找可安排空间，优先避开复习高峰。
6. 对待补学任务设置每日新增补学上限，使后续日期增加量尽量接近。
7. 每次放入一个待补学任务后更新该日负荷，避免同一天持续吸收任务。
8. 如果所有日期都达到硬上限仍无法安排，计划进入无法按期完成状态。

算法不得出现：

- 明天直接吞下今天全部缺口。
- 同一个未完成单词被重复生成多次。
- 因重排删除 `missed` / `overdue` 历史。
- 延长截止日期或减少总目标。

## 复习高峰避让

复习高峰日指以下任一情况：

- 计划复习数接近每日复习上限。
- 总负荷已达到偏高。
- 已存在逾期复习。

待补学新词优先安排到非高峰日。如果所有日期都有复习压力，则仍按总负荷最低优先，而不是机械平均。

## 不可行判定

以下情况判定为无法按期完成：

- 剩余有效学习日为 0 且目标未完成。
- 剩余新词数除以剩余学习日后的最低日均新词量超过每日新词硬上限。
- 待补学新词无法在未来有效学习日的剩余新词容量内放完。
- 复习和逾期复习无法在未来有效学习日的复习容量内放完。
- 词表缺口导致没有足够真实词条生成具体任务；该情况必须标为词表问题，不得记为用户未完成。

不可行解释必须包含：

- 剩余新词任务数。
- 剩余复习压力。
- 剩余有效学习日。
- 按期完成所需最低每日新词量。
- 当前每日新词上限。
- 当前每日复习压力。
- 无法完成的具体原因。

示例：

```text
剩余新词：1000
剩余学习日：9
按期完成至少需要每天 112 个新词
当前每日新词上限：105
当前设置下无法按期完成
```

## 调整原因

调整原因面向用户显示，不展示内部枚举。推荐表达：

- 今天没完成的 100 个新词已安排到后面的 9 个学习日。
- 后面几天每天大约多 11 到 12 个新词。
- 2026-06-05 复习较多，系统减少了当天补学任务。
- 当前上限不足，按期完成至少需要每天 112 个新词。
