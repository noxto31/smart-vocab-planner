# v0.3.0 用户流程冻结

## 页面职责

| 页面 | 核心职责 |
| --- | --- |
| 今日任务 / 首页 | 今日新词、今日复习、待补学、逾期、结算入口、最近调整 |
| 目标与阶段计划 | 创建目标、自然语言目标、建议预览、阶段路径、目标历史 |
| 词书与词库 | 导入词表、启用范围、去重统计、缺口、推荐组合 |
| 长期计划 | 阶段、月度、周度、每日计划及可行性状态 |
| 学习历史 | 具体词条状态、复习记录、重点遗忘词 |
| 复盘与调整 | 周度复盘、月度复盘、调整日志、AI 诊断建议 |
| 设置与数据 | 时区、容量参数、AI 配置状态、备份与恢复 |

## 首次进入软件

1. 软件展示当前没有执行目标和可用词库。
2. 首页提示先创建目标或载入演示数据。
3. 推荐词书只显示为推荐或候选，不生成具体任务。
4. 用户可以导入 CSV/JSON 或使用演示词表。

## 创建结构化目标

用户填写：

- 目标类型、目标说明。
- 开始日期、截止日期。
- 目标需求词量、当前基础说明、是否补基础词。
- 每日新词上限、每日复习上限。
- 每周学习日、固定休息日、缓冲日比例。
- 计划风格、时区、是否启用 AI 辅助。

保存后，本地校验目标和容量，生成目标版本记录和未来计划。

## 输入自然语言目标

1. 用户输入自然语言目标。
2. AI 服务接口或本地规则生成结构化建议。
3. 页面展示识别目标类型、阶段、词量、每日任务范围、词书组合、缺口和理由。
4. 用户未确认前，不改变正式目标、词书范围、具体任务和历史。
5. 用户确认后，创建目标版本历史并重新排期。

## 词书选择与导入

1. 词书页展示推荐词书、已导入词书、已启用学习词书、候选补充词书和已停用词书。
2. 用户导入词表后，系统显示新增词、重复词、补足缺口和剩余缺口。
3. 用户启用词书后，未来任务只从已启用且有具体词条的范围选择。
4. 已导入但未启用词书不自动参与排期。

## 生成计划

目标、词书范围和容量确认后，系统生成：

- 阶段计划。
- 长期总体状态。
- 月度计划。
- 周度计划。
- 每日具体新词任务和复习任务。

周月数据必须由每日具体任务汇总，不能脱离 `wordId` 任务单独估算。

## 执行今日新词学习

1. 首页显示今日任务数量、已完成数量、未处理数量、待补学、逾期和缺口。
2. 用户进入新词学习，逐词查看单词、释义、来源词书、阶段或标签、当前状态。
3. 支持“已学习 / 已掌握 / 暂时跳过 / 今日未完成”。
4. 逐词操作只影响当前词条及直接复习安排，同日其他开放任务保持开放。

## 执行今日复习

1. 用户进入复习列表，先看单词，点击查看释义。
2. 支持“不认识 / 模糊 / 认识 / 很熟悉 / 未完成”。
3. 页面展示首次学习日期、复习次数、最近结果、下一次复习日期、遗忘次数和掌握状态。
4. 单个复习结果不得提前将同日其他开放复习任务判为逾期。

## 词条详情与学习历史

学习历史页或侧栏展示：

- 单词、释义、来源词书与标签。
- 首次学习时间、所有复习历史、当前复习阶段、下一次复习日期。
- 遗忘次数、重点遗忘词标识。
- 当前是否待补学、逾期、复习中或已掌握。

## 结束今日学习并结算

1. 用户点击结算按钮。
2. 系统显示今日仍未处理新词和复习数量。
3. 用户确认后，开放新词转为待补学，开放复习转为逾期。
4. 本地调度器重排未来具体任务并写入调整日志。
5. 重复结算同一天必须幂等。

## 跨日自动结算

打开软件或加载数据时，系统按目标时区查找早于今天的开放任务：

- 历史开放新词转为待补学。
- 历史开放复习转为逾期。
- 今天的开放任务保持开放。
- 结算和重排必须幂等。

## 动态调整与复盘

每日结算、目标变化、词书导入或 AI 建议确认后，系统显示：

- 触发原因。
- 调整前后关键数量。
- 受影响日期。
- 目标和词书范围是否变化。
- 当前是否可按期完成。

周度复盘和月度复盘必须显示计划量、实际量、欠缺、逾期、掌握、重点遗忘词和后续负荷变化。

## 修改目标、容量或词书范围

1. 用户修改参数或启用词书。
2. 系统展示影响预览。
3. 用户确认后，创建目标版本或调整记录。
4. 真实学习历史、复习历史和 legacy 历史保留。
5. 仅未来计划重排。

## 导出与恢复

- 导出包含目标、目标历史、阶段、词书、词条、任务、历史、结算、复盘、AI 建议、调整日志和迁移元数据。
- 导入 v0.1.0 备份时，只保留数量记录为 legacy 历史。
- 导入 v0.2.0 / v0.2.1 备份时，兼容读取并迁移到 v0.3.0。
- 异常备份安全失败，不清空当前数据。
