# v0.3.0 产品范围：完整智能背词 Beta 版

## 版本定位

v0.3.0 直接承接 `codex/v0.2.1-daily-settlement-fix` 的有效代码、测试和数据兼容能力，目标是把软件从“可执行的单词级背词闭环”升级为“以动态学习计划为中枢的个人智能背单词 Beta 版”。

本版本定位：

> 一个以动态学习计划为中枢、支持真实背词执行、支持 AI 辅助目标理解和词书匹配、能够依据实际学习进度持续重排任务的个人智能背单词软件 Beta 版本。

## 要解决的真实问题

- 用户只有模糊目标时，不知道应先补基础、直接冲目标词书还是混合推进。
- 目标词量常常大于当前真实词库，旧计划容易把词库缺口误判为用户失败。
- 逐词背诵、复习、历史、重排和周月复盘需要共享同一套可测试的本地数据。
- AI 建议需要提供解释和方向，但不能绕过本地规则直接改计划。
- 用户需要能导出、恢复和迁移旧数据，并保留无法证明的旧历史事实边界。

## v0.2.1 承接关系

必须完整承接：

- 具体 `wordId` 级新词任务和复习任务。
- 单词逐项操作与当日结算分离。
- 主动结算和跨日自动结算的幂等处理。
- 词库供给缺口、学习欠缺和逾期复习的严格区分。
- v0.2.0 备份读取兼容。
- v0.1.0 legacy 数量记录迁移且不伪造具体单词历史。
- 浏览器本地时区和 Asia/Tokyo 日期边界测试。

## 必须完成的功能

P0：

- 结构化目标创建和修改。
- 自然语言目标建议、预览、确认后应用。
- 目标版本历史和计划调整日志。
- 词书推荐、导入、启用、停用和去重覆盖统计。
- 真实词库不足时只生成可执行具体任务，并保留供给缺口。
- 今日任务工作台、新词学习、今日复习和主动结算。
- 词条详情、复习历史和重点遗忘词规则标识。
- 长期、阶段、月度、周度、每日计划联动。
- 周度复盘、月度复盘和风险解释。
- 本地规则 AI 降级架构，建议必须本地校验并由用户确认。
- v0.3.0 备份导出、恢复、旧版迁移测试。

P1：

- 可解释的词书组合推荐角色：基础补齐、目标核心、强化扩展、冲刺高频。
- 具体学习范围选择优先级，不再无说明地按字母截取。
- AI 建议影响预览。
- 调整原因的用户可读解释。

P2：

- 真实 AI 服务配置入口只作为可选接口预留；若无法保证安全保存 API Key，则不宣称真实服务可用。
- 更丰富的统计筛选和页面细节。

## 明确不完成

- 账户注册、登录、多端云同步。
- 社交打卡、排行、游戏化系统。
- 自动抓取商业完整词书。
- 发音播放、在线例句、拼写测试、听力和口语训练。
- 应用商店发布。

## 正式发布判定

满足以下条件后才可进入发布尝试：

- v0.3.0 设计文档与实现一致。
- `npm.cmd test` 通过。
- `npm.cmd run build` 通过。
- v0.2.1 回归测试未退化。
- 旧版迁移和 v0.3.0 备份恢复有测试证据。
- README、CHANGELOG、WORKLOG、TASK_FULL 和 Release Notes 只记录真实实现。
- 推送 v0.3.0 开发分支；具备权限时再合并、tag 和创建 GitHub Release。
