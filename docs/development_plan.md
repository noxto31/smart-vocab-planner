## 阶段 1：MVP 实现

实现内容：

1. 初始化 React + TypeScript + Vite 项目。
2. 建立 `src/domain/`，放置类型、日期工具、导入解析、推荐和调度算法。
3. 建立 `src/storage/`，使用 Dexie 管理 IndexedDB。
4. 建立 `src/services/`，封装应用状态、演示数据、备份导入导出和计划持久化。
5. 建立中文界面，包含总览、目标设置、今日任务、计划查看、词书管理、统计、数据管理。
6. 建立 `sample-data/`，提供演示词书、演示 CSV、重复导入示例、导入模板和演示备份。

## 阶段 2：测试与验证

自动化测试覆盖：

- 正常生成长期、月度、周度和每日计划。
- 1000 个待学习词、100 个有效学习日时平均约 10 个新词。
- 休息日不安排任务。
- 缓冲日不会在任务量允许时被主要新学任务占满。
- 少完成任务后，未来任务吸收欠缺量。
- 超额完成后，剩余任务减少。
- 连续多天未完成时总目标不减少。
- 不可行状态显示最低每日任务量和上限差距。
- 每日新学上限生效。
- 多词书导入去重并保留来源。
- 导出后重新导入能恢复目标和完成记录。

## 质量检查

完成代码后运行：

```bash
npm.cmd test
npm.cmd run build
```

若发现问题，先修复核心逻辑和类型错误，再更新 `README.md` 与 `WORKLOG.md`。

## 已知取舍

- 第一版记录每日完成数量，不制作逐词背诵卡片。
- 复习任务按数量调度，保留 `ReviewTask` 模型供后续扩展到单词级。
- 词书推荐使用可解释规则，不联网检索词库。
- 内置词表仅为演示和测试，不代表任何商业词书完整内容。
