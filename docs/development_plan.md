# 开发计划与测试范围

## v0.1.0 基线

- 已将第一版定义为“离线数量计划 MVP 基线版”。
- 已补充 `CHANGELOG.md` 的 v0.1.0 记录。
- 已创建并推送 `v0.1.0` tag。
- GitHub Release 因当前环境缺少 `gh` CLI，未创建。

## v0.2.0 实施内容

1. 升级数据模型：新增 `WordProgress`、`DailyNewWordAssignment`、`DailyReviewAssignment`、`PlanCoverageStatus` 和新版调整日志。
2. 升级 Dexie：使用 v2 表结构，迁移 v0.1.0 旧目标和旧数量记录。
3. 重构调度：从数量任务升级为具体单词任务，区分库存缺口和学习欠缺。
4. 增加复习闭环：新学后生成复习，到期反馈后决定下一次复习。
5. 增加 AI mock：自然语言目标入口、结构化建议和用户确认流程。
6. 更新 UI：总览、目标设置、词书管理、今日学习、计划、统计、数据管理。
7. 更新测试、README、CHANGELOG、WORKLOG 和验收文档。

## 测试范围

- 具体单词任务生成。
- 多词书去重与来源保留。
- 词库供给缺口不计为用户欠缺。
- 用户未完成任务进入待补学重排。
- 导入新词书补足库存缺口。
- 容量不足不可行判断。
- 复习反馈与复习历史。
- v0.1.0 备份兼容迁移。
- Asia/Tokyo 日期边界。

## 发布限制

v0.2.0 代码只推送到 `codex/v0.2.0-learning-loop` 并创建 PR。正式 tag 和 Release 等待验收合并后执行。
