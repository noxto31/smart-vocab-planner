# v0.2.0 验收说明

## 场景一：词库不足但用户完成全部可执行任务

操作：

1. 创建目标：目标需求量 300，截止日期为今天，每日新词上限 300。
2. 导入 70 个去重词。
3. 生成计划。
4. 在“今日学习”把 70 个新词全部标记为“已学习”。

预期页面结果：

- 今日可执行具体新词全部完成。
- 词库供给缺口显示 230。
- 学习欠缺任务显示 0。
- 调整说明提示库存缺口不计为用户未完成。

预期数据状态：

- `availableWordCount = 70`
- `inventoryGapCount = 230`
- `completedWordCount = 70`
- `learningBacklogCount = 0`

自动化测试：

- `tests/scheduler.test.ts`：`场景一：词库 70 词、目标 300 词...`

## 场景二：词库足够但用户未完成

操作：

1. 创建目标：目标需求量 300，已有 300 个去重词，每日新词上限 300。
2. 生成当天 300 个具体新词任务。
3. 将 70 个标记为“已学习”，其余标记为“今日未完成”。

预期页面结果：

- 词库供给缺口为 0。
- 学习欠缺任务为 230。
- 230 个具体单词被保留并重排到后续日期。

预期数据状态：

- `inventoryGapCount = 0`
- `completedWordCount = 70`
- `learningBacklogCount = 230`
- 后续存在 `status = rescheduled` 的 `DailyNewWordAssignment`。

自动化测试：

- `tests/scheduler.test.ts`：`场景二：词库足够但用户只完成 70 个时...`
- `tests/scheduler.test.ts`：`后续容量不足时显示不可行...`

## 场景三：导入新词书后补足库存缺口

操作：

1. 在存在 230 个库存缺口的目标下导入一份新增 180 个去重词的词表。
2. 系统自动重排。

预期页面结果：

- 词库供给缺口从 230 降低到 50。
- 新增词生成后续具体新词任务。
- 调整日志记录新增、补足和剩余缺口。

预期数据状态：

- `availableWordCount = 250`
- `inventoryGapCount = 50`
- 新增 180 个 `DailyNewWordAssignment`

自动化测试：

- `tests/scheduler.test.ts`：`场景三：导入新增 180 个去重词后...`
