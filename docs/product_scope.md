# 产品范围：智能背单词计划安排器 MVP

## 产品定位

第一版是本地离线的背单词目标规划与动态调度工具，核心价值是帮助用户设定总目标、导入或选择词书、生成多层级学习计划，并在实际完成不足或超额时重新分配后续任务。

本 MVP 不实现完整背词训练流程，不提供发音、例句生成、拼写训练、云同步、登录、社交分享、外部大模型 API 或联网抓取词库。

## 核心用户流程

1. 用户载入演示词书或导入 CSV/JSON 词表。
2. 用户填写学习目标、截止日期、每日新学上限、每日复习上限、休息日、缓冲日比例和计划类型。
3. 系统根据实际去重后的目标词汇量生成长期、月度、周度和每日计划。
4. 用户在今日任务页记录当天新学、复习和学习时长。
5. 系统保留历史完成记录，重新计算未来任务分布。
6. 当剩余任务无法在每日上限和截止日期内完成时，系统显示不可行状态和原因。
7. 用户可以导出完整本地数据，也可以导入备份恢复目标、词表和完成记录。

## 页面结构

- 总览仪表盘：目标、进度、今日任务、周/月摘要、计划状态。
- 目标设置页：学习目的、词汇目标、日期、容量限制、休息日、计划风格。
- 今日任务页：今日计划任务、完成记录表单、最近调整说明。
- 计划查看页：长期指标、月度计划、周计划、每日计划列表。
- 词书管理页：推荐词书、演示数据、CSV/JSON 导入、去重统计。
- 统计页：累计完成、欠缺任务、每日/周/月完成率、当前所需平均任务量。
- 设置与数据管理页：修改限制、导出备份、导入备份、重置数据。

## MVP 功能边界

必须实现：

- 本地 IndexedDB 持久化。
- 词书元数据管理。
- CSV 和 JSON 词表导入。
- 单词标准化、去重和来源保留。
- 用户目标配置和校验。
- 长期、月度、周度、每日计划生成。
- 日完成记录。
- 动态重排未来任务。
- 不可行判断。
- 数据导入导出。
- 自动化测试。

暂不实现：

- 按单词卡片逐词学习。
- 精细到单个单词的完整复习 UI。
- 用户登录、云同步、多端同步。
- 商业词书全文抓取或内置。
- 外部模型生成内容。

## 验收标准

- `npm.cmd run dev` 可以启动本地 Web 应用。
- 用户可以载入演示数据、设置目标并生成计划。
- 用户可以查看今日、每周、每月和长期计划。
- 用户可以记录今日完成情况，系统会重新分配未来任务。
- 动态调整不会修改总目标、截止日期、词书范围或历史完成记录。
- 不可行目标会显示剩余任务、剩余学习日、最低每日新学量和与上限的差距。
- `npm.cmd test` 覆盖核心算法和导入导出逻辑。
- `npm.cmd run build` 可以完成生产构建。
