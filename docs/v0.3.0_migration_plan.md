# v0.3.0 迁移计划

## 迁移目标

v0.3.0 将 Dexie 数据结构升级到 v3，同时继续支持 v0.1.0、v0.2.0 和 v0.2.1 备份读取。迁移必须保留真实历史，不创建无法证明的事实。

## v0.1.0 legacy 数据

- 目标字段迁移为当前目标结构。
- 词表按已有词条迁移。
- 旧 `progressRecords` 只进入 `legacyProgressRecords`。
- 不创建 `wordProgress`、`dailyNewAssignments`、`dailyReviewAssignments` 或 `reviewHistory`。
- 页面显示“旧版本只记录数量，无法可靠转换为具体单词历史”。

## v0.2.0 开发版备份

- 识别 `schemaVersion: 2` 和 `backupVersion: v0.2.0`。
- 保留已有具体任务、单词状态、复习历史和调整日志。
- 新增 v0.3.0 表用空数组或从真实任务汇总生成。
- 不因开发版缺少字段而拒绝导入，只对必要数组字段做校验。

## v0.2.1 技术基线数据

- 直接迁移 v2 表到 v3。
- 保留逐词操作和当日结算修复产生的数据形态。
- 基于现有目标创建初始目标版本记录。
- 基于现有每日任务汇总生成周度和月度复盘初始记录。
- 已有调整日志保留。

## v0.3.0 新增字段和表

新增或扩展：

- 目标版本历史。
- 阶段计划。
- 词书状态、启用状态和推荐角色。
- 每日结算记录。
- 周度复盘记录。
- 月度复盘记录。
- AI 建议记录和应用记录。
- 计划调整日志扩展字段。
- 备份迁移元数据。

## 备份导入安全

导入流程：

1. 先解析并校验备份。
2. 在内存中完成迁移。
3. 关键结构校验通过后才清空当前数据。
4. 使用单次事务写入。
5. 任一步失败时保留当前数据库。

## 不得伪造的数据

- 旧数量记录不得映射到随机词。
- 未记录过的复习结果不得补造。
- 未导入具体词条的推荐词书不得生成任务。
- 缺失的 AI 建议记录不得虚构。

## 验证要求

- v0.1.0 迁移不产生具体任务。
- v0.2.0 备份可读取。
- v0.2.1 数据升级后任务、历史和日志保持。
- v0.3.0 导出后可重新导入。
- 异常备份安全失败，不破坏当前数据。
