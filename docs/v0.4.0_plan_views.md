# v0.4.0 计划视图说明

## 目标

v0.4.0 将长期表格从主视图降级为详细明细，把日常入口改为更容易理解的计划视图：

- 今日任务卡片。
- 周计划卡片。
- 月历 / 负荷热力图。
- 阶段时间轴。
- 详细明细。

每个视图必须回答一个明确的用户问题，而不是暴露数据库字段。

## 今日任务卡片

回答的问题：

```text
我今天到底要学什么？为什么比原计划多或少？压力大不大？
```

必须显示：

- 今日原计划。
- 今日调整后任务。
- 今日新词。
- 今日复习。
- 今日待补学。
- 今日逾期复习。
- 今日总负荷。
- 任务压力：轻松 / 正常 / 偏高 / 超限 / 休息。
- 调整原因。
- 完成进度。
- 结束今日学习入口。

示例文案：

```text
今日任务：新词 112 个，复习 80 个
比原计划多 12 个
原因：昨天少完成 100 个新词，系统已分摊到后续 9 天
当前压力：偏高，但仍在上限内
```

## 周计划卡片

回答的问题：

```text
接下来 7 天任务是否平稳？哪几天变多了？
```

每张日卡显示：

- 日期和星期。
- 新词任务。
- 复习任务。
- 待补学。
- 逾期复习。
- 总负荷。
- 压力状态。
- 是否休息日。
- 是否因动态调整发生变化。
- 调整说明。

周视图必须能看出：

- 哪几天任务增加。
- 增加原因。
- 哪天压力偏高。
- 哪天是休息日或缓冲日。
- 补学任务是否平滑分配。

## 月历 / 负荷热力图

回答的问题：

```text
这个月的学习压力分布是否健康？哪里有高峰？
```

每个日期格子至少显示：

- 日期。
- 新词数量。
- 复习数量。
- 总负荷。
- 压力颜色或状态。
- 是否已完成。
- 是否未完成。
- 是否休息日。
- 是否有待补学或逾期复习。

压力状态使用统一中文：

- 休息。
- 轻松。
- 正常。
- 偏高。
- 超限。
- 已完成。
- 未完成。

热力图可以用浅色到深色表现压力，但不得只输出 160 行日期表。

## 阶段时间轴

回答的问题：

```text
我现在处于计划的哪一段？这一段还有多少？
```

每个阶段显示：

- 当前目标。
- 当前阶段。
- 阶段名称。
- 阶段日期范围。
- 阶段目标词数。
- 阶段已完成词数。
- 阶段剩余词数。
- 阶段状态。
- 是否因动态重排发生变化。
- 阶段与词书或词条范围的绑定说明。

如果阶段名称来自本地计划规则，而不是完整真实词书范围，必须提示：

```text
当前阶段名称基于计划规则生成，尚未绑定完整真实词书。
```

## 详细明细

回答的问题：

```text
我想核对每一天的具体数量或排查问题。
```

详细明细保留每日表格，但必须作为辅助入口：

- 名称建议为“详细明细”。
- 可以折叠或放在计划页底部。
- 可以用于展开全部日期、导出或调试。
- 不再作为主计划视图。

## 数据来源

视图数据应由领域层或轻量视图模型函数生成，界面只负责展示：

- 今日任务卡片读取今日 `DailyTaskSummary` 和对应任务。
- 周计划卡片读取连续 7 天聚合。
- 月历 / 热力图按月份聚合每日负荷。
- 阶段时间轴读取 `StagePlan` 并补充真实完成进度。
- 详细明细读取每日任务。
