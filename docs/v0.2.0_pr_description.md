# PR Description Draft

## 本轮版本目标

将 v0.1.0 的数量计划 MVP 升级为 v0.2.0 智能背词主流程与动态规划闭环版。

## 与 v0.1.0 的差异

- v0.1.0 只安排数量；v0.2.0 安排具体单词。
- v0.1.0 只记录每日完成数量；v0.2.0 记录具体新词学习和具体复习结果。
- v0.1.0 未严格区分库存缺口和学习欠缺；v0.2.0 使用 `PlanCoverageStatus` 明确区分。

## 主要新增功能

- 具体单词级新词任务。
- 具体单词级复习任务和复习历史。
- 词库供给缺口、学习欠缺任务和逾期复习统计。
- 自然语言目标入口和 AI mock 建议。
- 导入新词表后补足库存缺口并重排。
- v0.2.0 完整备份和 v0.1.0 备份迁移。

## 数据模型变化

- 新增 `WordProgress`。
- 新增 `DailyNewWordAssignment`。
- 新增 `DailyReviewAssignment`。
- 新增 `ReviewHistoryRecord`。
- 新增 `PlanCoverageStatus`。
- 升级 `PlanAdjustmentLog` 为调整前后快照。

## 数据迁移处理

- Dexie 升级到 v2。
- v0.1.0 目标迁移到 `LearningGoal`。
- v0.1.0 数量完成记录迁移为 `legacyProgressRecords`，不伪造具体单词历史。
- v0.2.0 备份包含 `backupVersion: "v0.2.0"`。

## 三个核心验收场景

1. 目标 300，词库 70，用户学完 70：库存缺口 230，学习欠缺 0。
2. 目标 300，词库 300，用户只完成 70：库存缺口 0，学习欠缺 230，具体词重排。
3. 存在库存缺口时导入新增 180 词：缺口降低并生成后续具体任务。

自动化测试位置：`tests/scheduler.test.ts`。
