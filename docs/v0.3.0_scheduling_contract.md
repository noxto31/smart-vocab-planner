# v0.3.0 调度行为契约

## 总原则

- 调度逻辑必须在 `src/domain` 中可单元测试调用，界面只调用服务层。
- 每个新词任务和复习任务必须绑定 `wordId`。
- 重排未来计划不得删除真实学习历史和复习历史。
- AI 输出只能作为建议输入，最终排期由本地调度器执行。

## 目标需求量

目标需求量来自当前已确认目标版本：

- 结构化目标直接使用 `targetRequiredCount`。
- 自然语言目标必须先生成建议，用户确认后才形成执行目标。
- 修改目标后创建新的目标版本记录。

## 真实词库供给量

真实供给量只统计数据库中存在的去重 `WordItem`：

- `availableWordCount`：全部导入去重词。
- `enabledWordCount`：当前目标已启用词书中的去重词。
- `inventoryGapCount = max(0, targetRequiredCount - enabledWordCount)`。

推荐但未导入的词书不计入可执行供给。

## 词书启用

- 已导入但未启用的词书不进入当前目标排期。
- 推荐词书只用于解释和补缺建议。
- 停用词书不再生成未来新词任务，但历史任务和复习历史保留。

## 具体新词选择

选择顺序必须可解释：

1. 排除用户已排除词。
2. 优先当前目标启用词书。
3. 按词书启用优先级和阶段角色排序。
4. 基础补齐阶段优先基础词书。
5. 目标核心阶段优先目标词书。
6. 已学、已掌握词不重复生成新词任务。
7. 数据不足时使用明确规则优先级，不伪造频率或考试权重。

## 词库缺口与补足

- 目标可以大于当前供给。
- 当前无真实词条时，不生成具体学习任务。
- 用户完成全部当前可执行词条后，缺口仍是 `inventoryGapCount`，不是 `learningBacklogCount`。
- 导入新词书后，按新增去重词补足缺口并重新排期。

## 学习欠缺

学习欠缺只来自：

- 用户明确标记新词“暂时跳过”或“今日未完成”。
- 用户主动结算当日开放新词。
- 跨日自动结算历史开放新词。

调度器不得仅因 `planned` 任务日期早于 `asOfDate` 就把它转为欠缺，必须走结算流程。

## 复习任务

- 学习新词后根据复习间隔创建具体复习任务。
- 复习结果更新 `WordProgress`、写入 `ReviewHistoryRecord`，并生成下一次复习或掌握状态。
- 逾期复习只来自用户明确未完成、主动结算或跨日自动结算。
- 同日其他开放复习不得因单个复习结果提前逾期。

## 逐词操作与当日结算

- 单个新词结果只更新该任务、该词进度和直接后续复习。
- 单个复习结果只更新该复习任务、复习历史和下一次复习。
- 今日其他开放任务通过 `preserveOpenDates` 保持开放。
- 主动结算后，剩余开放任务统一进入待补学或逾期。
- 重复结算必须返回无变化结果。

## 跨日自动结算

加载数据时按目标时区处理：

- 只结算早于目标时区今日的开放任务。
- 今天和未来开放任务不结算。
- 多目标逐一处理。
- 多次执行结果一致。

## 周度与月度复盘

- 周度和月度复盘由每日任务、学习历史、复习历史和调整日志汇总。
- 当周完成不足时，未来任务重排并记录原因。
- 当月复习压力超容量时，风险状态必须显示数量依据。

## 容量与不可行判断

不可行条件包括：

- 剩余有效学习日为 0 且目标未完成。
- 按期完成所需最低每日新词量超过每日上限。
- 逾期复习和计划复习超过每日复习上限且无法排入剩余日期。
- 真实词库供给缺口导致当前阶段无法生成具体任务。

风险说明必须包含剩余具体单词、缺口、待补学、逾期、剩余有效学习日和当前上限。

## 目标修改

目标、容量、期限或词书范围变化后：

1. 保存目标版本历史。
2. 保留真实学习和复习历史。
3. 仅重排未来任务。
4. 写入计划调整日志。
5. 显示调整前后任务量和可行性。
