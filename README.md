# 智能背单词计划安排器

当前版本：v0.2.0 - 智能背词主流程与动态规划闭环版

这是一个本地离线 Web 应用，核心定位是“真正可背词的软件 + 动态计划中枢 + AI 规划接口”。用户可以导入真实词表，生成绑定具体单词的新学任务，逐词学习和复习，并让本地调度逻辑根据词库缺口、学习欠缺和复习压力持续重排计划。

## 技术栈

- React + TypeScript + Vite
- Dexie / IndexedDB
- Vitest
- 本地 mock AIPlanningService，不接入真实外部 API

## 启动方式

```bash
npm.cmd install
npm.cmd run dev
```

运行测试和构建：

```bash
npm.cmd test
npm.cmd run build
```

## 主要使用流程

1. 在“词书管理”载入演示词表，或导入 CSV/JSON 词表。
2. 在“目标设置”填写结构化目标，或输入自然语言目标并生成本地 AI mock 建议。
3. 保存目标后，系统生成绑定具体 `wordId` 的每日新词任务。
4. 在“今日学习”逐词标记“已学习 / 已掌握 / 暂时跳过 / 今日未完成”。
5. 到期复习时先查看单词，再查看释义，并选择“不认识 / 模糊 / 认识 / 很熟悉 / 未完成”。
6. 系统根据真实学习记录、复习结果、词库供给缺口和每日容量重排后续任务。
7. 在“计划”和“统计”查看长期覆盖、月/周/日计划、欠缺任务、逾期复习和调整日志。
8. 在“数据管理”导出或导入完整备份。v0.1.0 备份会兼容导入为历史数量记录。

## 导入词表格式

CSV 字段：

```csv
word,meaning,book_name,level,tags
example,例子,My Word Book,B1,noun
adapt,适应,My Word Book,B1,verb;core
```

JSON 顶层应为数组，每项至少包含 `word` 字段，可选 `meaning`、`book_name`、`level`、`tags`。

同一单词来自多本词书时，系统只创建一个学习对象，但保留多个来源词书。

## 当前已实现

- 具体单词级新词任务。
- 具体单词级复习任务、反馈和复习历史。
- 词库供给缺口与学习欠缺任务分开统计。
- 导入新词书后补足词库缺口并重排。
- 自然语言目标入口与 AI mock 建议。
- Dexie v2 数据结构和 v0.1.0 备份兼容迁移。
- 默认使用浏览器本地时区，目标中保存计划时区。
