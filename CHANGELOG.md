# Changelog

## v0.4.0 - 动态进度恢复与计划视图重构版

### 主要目标

- 重构动态进度恢复机制。
- 增加日历、周视图、负荷热力图和阶段时间轴。
- 统一中文产品用词，减少开发术语。

### 已完成

- 新增基于总负荷的动态恢复分配：新词任务与复习压力共同参与计算，复习权重为 `0.6`。
- 漏学一天或少完成一部分后，未完成的具体单词会平滑分摊到后续有效学习日。
- 复习高峰日会降低补学分配优先级。
- 每日摘要新增原计划、调整后任务、待补学、总负荷、舒适负荷、硬上限和任务压力。
- 新增今日任务卡片、周计划卡片、月历 / 负荷热力图、阶段时间轴和详细明细视图模型。
- 建立统一中文状态映射，页面使用“本期目标词数”“目标词表缺口”“待补学”“任务压力”等表达。
- 修正本地规则式自然语言目标建议：无真实目标词表时，不生成六级 900 词这类正式精确建议。
- v0.4.0 备份导出标记为 `backupVersion: "v0.4.0"`，继续读取 v0.3.0 schema 3 备份。

### 测试

- 新增动态恢复测试，覆盖漏学一天、部分少学、复习高峰避让和每日上限不足。
- 新增计划视图测试，覆盖今日卡片、周计划、月历热力图、阶段时间轴和详细明细。
- 新增中文用词与六级建议测试。

### 不包含

- 不扩充正式词书库。
- 不接入真实 AI API。
- 不新增发音、例句、拼写训练。
- 不新增登录、云同步或应用商店发布。

## v0.3.0 - 智能背词规划与数据一致性正式版

### 版本定位

- 直接承接未正式发布的 v0.2.1 技术基线分支 `codex/v0.2.1-daily-settlement-fix`。
- 完整纳入 v0.2.1 对逐词操作与当日结算一致性的修复。
- 在此基础上升级目标规划、词书匹配、学习执行、动态计划、复盘和 AI 辅助结构。

### 设计冻结

- 新增 v0.3.0 产品范围、用户流程、数据模型、调度契约、AI 契约、迁移计划、验收标准和实施计划文档。
- 明确推荐词书、已导入词书、已启用词书、候选补充词书和已停用词书的状态边界。
- 明确 AI 默认以本地规则模式工作；任何 AI 建议必须本地校验并由用户确认后才可影响目标或计划。
- 明确旧版数量记录只作为 legacy 历史保留，不得映射为具体单词学习历史。

### 主要新增

- Dexie v3 数据结构，新增目标版本历史、阶段计划、每日结算记录、周度复盘、月度复盘、AI 建议记录和 AI 应用记录。
- v0.3.0 备份格式，导出目标历史、阶段计划、词书状态、具体任务、复习历史、复盘、AI 建议和迁移元数据。
- 自然语言目标本地规则解析，建议在确认前不改变正式目标或任务，确认后写入应用记录。
- 可解释词书推荐结构，区分推荐、已导入、已启用、候选补充和停用状态。
- 具体单词选择优先级从单纯字母排序升级为词书启用顺序、阶段角色、基础补齐、核心标签和学习状态的综合规则。
- 今日任务、目标与阶段、词书与词库、长期计划、学习历史、复盘与调整、设置与数据的页面结构整理。
- 词条学习历史、复习历史和重点遗忘词规则标识。
- 周度复盘、月度复盘和风险数量解释。
- 异常备份导入在解析失败时不会清空当前数据。
- `availableWordCount` 与 `enabledWordCount` 分离：可用词量统计全部可用词，启用词量只统计当前目标纳入计划的词。
- AI 建议应用记录与普通手动修改分离：只有显式应用建议才新增 AI 应用记录。
- 新增 `.github/workflows/release-on-tag.yml`，推送版本 tag 后自动验证并创建 GitHub Release。

### 关键承接修复

- 逐词操作不会提前结算同日其他任务。
- 主动结算与跨日自动结算保持可靠。
- 词库供给缺口、学习欠缺和逾期复习继续严格区分。
- 旧版备份与 legacy 数据继续兼容。

### 暂未包含

- 真实 AI API 默认接入；当前为本地规则模式和可接入服务结构。
- 发音、例句、拼写、听力、登录和云同步。
- 商业完整词书自动抓取。

## v0.2.1 - 未正式发布的技术基线版本

- 修复逐词操作与当日结算一致性问题。
- 增加服务层测试，覆盖单个新词、单个复习、主动结算、跨日自动结算、幂等性和旧版备份兼容。
- 其有效能力已由 v0.3.0 完整承接。
- 当前 Git 中实际存在 `v0.2.1` tag，但 GitHub Releases 页面无 v0.2.1 Release；本轮不再单独创建 v0.2.1 Release。

## v0.2.0 - 未正式发布的开发中间版本

- 初步建立单词级学习闭环。
- 支持具体新词任务、具体复习任务、词库缺口和本地 AI mock 入口。
- 后续发现存在逐词操作导致当日其他任务被提前重排的问题。
- 不创建正式 Release。

## v0.1.0 - 离线数量计划 MVP 基线版

- 已作为早期正式基线保留。
- 实现目标设置、基础词表导入、数量级计划、每日完成记录、动态重排、可行性判断和备份恢复。
- 当前限制是没有具体单词级任务、没有真实复习闭环，词库不足与用户未完成任务尚未严格区分。
