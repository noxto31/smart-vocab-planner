# Changelog

## v0.3.0 - 完整智能背词 Beta 版

### 版本定位

- 直接承接未正式发布的 v0.2.1 技术基线分支 `codex/v0.2.1-daily-settlement-fix`。
- 完整纳入 v0.2.1 对逐词操作与当日结算一致性的修复。
- 在此基础上升级目标规划、词书匹配、学习执行、动态计划、复盘和 AI 辅助结构。

### 设计冻结

- 新增 v0.3.0 产品范围、用户流程、数据模型、调度契约、AI 契约、迁移计划、验收标准和实施计划文档。
- 明确推荐词书、已导入词书、已启用词书、候选补充词书和已停用词书的状态边界。
- 明确 AI 默认以本地规则模式工作；任何 AI 建议必须本地校验并由用户确认后才可影响目标或计划。
- 明确旧版数量记录只作为 legacy 历史保留，不得映射为具体单词学习历史。

### 主要新增

- 待实施完成后按真实功能填写。

### 关键承接修复

- 逐词操作不会提前结算同日其他任务。
- 主动结算与跨日自动结算保持可靠。
- 词库供给缺口、学习欠缺和逾期复习继续严格区分。
- 旧版备份与 legacy 数据继续兼容。

### 暂未包含

- 真实 AI API 默认接入。
- 发音、例句、拼写、听力、登录和云同步。
- 商业完整词书自动抓取。

## v0.2.1 - 未正式发布的技术基线版本

- 修复逐词操作与当日结算一致性问题。
- 增加服务层测试，覆盖单个新词、单个复习、主动结算、跨日自动结算、幂等性和旧版备份兼容。
- 其有效能力已由 v0.3.0 完整承接。
- 当前 Git 中实际存在 `v0.2.1` tag，但 GitHub Releases 页面无 v0.2.1 Release；本轮不再单独创建 v0.2.1 Release。

## v0.2.0 - 未正式发布的开发中间版本

- 初步建立单词级学习闭环。
- 支持具体新词任务、具体复习任务、词库缺口和本地 AI mock 入口。
- 后续发现存在逐词操作导致当日其他任务被提前重排的问题。
- 不创建正式 Release。

## v0.1.0 - 离线数量计划 MVP 基线版

- 已作为早期正式基线保留。
- 实现目标设置、基础词表导入、数量级计划、每日完成记录、动态重排、可行性判断和备份恢复。
- 当前限制是没有具体单词级任务、没有真实复习闭环，词库不足与用户未完成任务尚未严格区分。
