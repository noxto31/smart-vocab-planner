# 智能背单词计划安排器

当前开发版本：v0.4.0 - 动态进度恢复与计划视图重构版

这是一个本地离线 Web 应用，核心定位是“真正可背词的软件 + 动态计划中枢 + 智能规划建议”。用户可以导入真实词表，生成绑定具体单词的新学任务，逐词学习和复习，并让本地调度逻辑根据待补学、逾期复习、词表缺口和任务压力持续调整学习安排。

v0.4.0 聚焦期限约束下的动态进度恢复：用户漏学或少背后，系统会保留具体未完成单词，并尽量把待补学平滑分摊到后续学习日，避开复习高峰；若当前上限不足，系统会明确说明无法按期完成的原因。产品边界见 `docs/v0.4.0_dynamic_recovery_algorithm.md`、`docs/v0.4.0_plan_views.md`、`docs/v0.4.0_language_guidelines.md` 和 `docs/v0.4.0_acceptance.md`。

## 技术栈

- React + TypeScript + Vite
- Dexie / IndexedDB
- Vitest
- 本地规则智能规划建议，不默认接入真实外部 API

## 主要使用流程

1. 在“词书管理”载入演示词表，或导入 CSV/JSON 词表。
2. 在“目标设置”填写结构化目标，或输入自然语言目标并生成本地 AI mock 建议。
3. 保存目标后，系统生成绑定具体 `wordId` 的每日新词任务。
4. 在“今日学习”逐词标记“已学习 / 已掌握 / 暂时跳过 / 今日未完成”。
5. 到期复习时先查看单词，再查看释义，并选择“不认识 / 模糊 / 认识 / 很熟悉 / 未完成”。
6. 逐词操作只影响当前词条；用户点击“结束今日学习并安排未完成任务”后，剩余未处理任务才会结算并重排。
7. 在“计划”查看今日卡片、周计划、月历热力图、阶段时间轴和详细明细。
8. 在“统计”查看学习历史、待补学、逾期复习和旧版历史数量记录。
9. 在“数据管理”导出或导入完整备份。v0.1.0 备份会兼容导入为历史数量记录。

## 导入词表格式

CSV 字段：

```csv
word,meaning,book_name,level,tags
example,例子,My Word Book,B1,noun
adapt,适应,My Word Book,B1,verb;core
```

JSON 顶层应为数组，每项至少包含 `word` 字段，可选 `meaning`、`book_name`、`level`、`tags`。

同一单词来自多本词书时，系统只创建一个学习对象，但保留多个来源词书。

## v0.4.0 已实现

- 动态恢复算法从按容量顺序填充改为按未来日期负荷平滑分配。
- 漏学一天后，待补学新词会分摊到剩余有效学习日，不会集中堆到第二天。
- 少完成一部分后，只把未完成的具体 `wordId` 纳入待补学。
- 复习高峰日会降低补学分配优先级。
- 新增总负荷模型：`新词 + 复习 * 0.6`，并输出轻松、正常、偏高、超限、休息。
- 不可行状态显示剩余任务、剩余学习日、最低日均新词和当前上限。
- 新增今日任务卡片、周计划卡片、月历 / 负荷热力图、阶段时间轴和详细明细。
- 统一中文产品用词，不在界面直接显示 `ok`、`near_limit`、`planned`、`backlog` 等内部状态。
- 六级等自然语言目标在没有真实目标词表时不再生成 900 词这类无依据精确建议。
