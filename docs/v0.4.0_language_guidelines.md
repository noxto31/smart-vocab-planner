# v0.4.0 中文产品用词规范

## 原则

- 面向用户说“学习安排”，不要说“排期”。
- 面向用户说“待补学”，不要说 `backlog`。
- 面向用户说“任务压力”，不要说 `capacityStatus`。
- 面向用户说“词表缺口”，不要说 `inventoryGap`。
- 状态翻译集中维护，页面不得散落写死内部枚举。
- 文案要短、清楚、能解释下一步。

## 禁止直接展示的内部术语

以下词不应裸露在页面中：

```text
inventoryGap
backlog
capacityStatus
planned
reviewing
mastered
active
ok
near_limit
over_limit
recommended
imported
enabled
candidate
disabled
```

如果文档或测试需要引用内部状态，应使用反引号并解释其中文含义。

## 推荐表达

不要写：

```text
当前词库供给不足，无法绑定计划量。
```

建议写：

```text
当前词表还不够，暂时只能安排已导入的单词。
```

不要写：

```text
学习欠缺任务已进入后续重排。
```

建议写：

```text
今天没完成的单词已安排到后面的学习日。
```

不要写：

```text
容量状态 near_limit。
```

建议写：

```text
今日任务偏多，但仍在可接受范围内。
```

## 六级目标建议

没有完整真实词表和基础测评时，不得生成看似精确的六级新增词数作为正式建议，例如“六级需要新增 900 词”。

推荐表达：

```text
还没有导入目标词表或完成基础评估，暂时无法生成可靠的新增词量。可以先导入目标词表，或使用演示词表体验计划流程。
```

如显示区间，必须说明：

```text
以下只是参考范围，不是已确认的学习量。
```
